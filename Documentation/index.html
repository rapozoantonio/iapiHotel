Alpha Hotel HTML Template
By Klayemore

Hey, thanks for purchasing my template. This documentation includes everything you should need to know. If you are happy with your purchase, I'd love it if you could help me out by rating my template. If you have any issues, please feel free to contact me through Themeforest and I'll help you out as best I can. Cheers!

This template is built with HTML5 and comes with 18 HTML/PHP files. The editable CSS file is 'style.css', the editable JS file is 'script.js' and the 'system' folder contains other template-specific files such as email scripts and language flags.

  • index.html - The default home page
  • accommodation.html - Displays all room types.
  • accommodation-room.html - The default room display.
  • location.html - For information about the location and local attractions.
  • gallery.html - A full-width, lazy load gallery.
  • about.html - General information about the hotel.
  • packages.html - Displays special packages for the hotel.
  • restaurant.html - Includes a menu, ideal for restaurants, day spas or wedding / conference venues.
  • testimonials.html - For guest testimonials.
  • blog.html - For blog articles or local events.
  • blog-post.html - Blog post detail view.
  • shortcodes.html - Displays all of the theme sections available.
  • contact.html - Contact page with form.

  • contact.php - As per contact.html, but with a functioning PHP Mailer form.
  • subscribe.php - A working example of the newsletter sign up form.
  • template-guest.php - Email template sent to guests when a user fills out the contact form.
  • template-hotel.php - Email template sent to the hotel when a user fills out the contact form.

There are index and contact pages ending in 'no-booking'. These are to be used for businesses that aren't hotels and don't need to take bookings. There is an additional 'no-booking' class added to the container and a slightly different navigation that will need to be replicated across all pages.

Styling

Header Logo

The logo has 2 instances - 'Dark' used as default, and 'Light' used when the booking panel is revealed. Be sure to replace both of the logos, or simply delete the 'Light' version if you are unsure how to invert the colours of your logo.

HTML5 Icons

All icons are being loaded from Ion Icons. To replace an icon, get the code and replace the relevant icon in the HTML. There is also a handy Cheatsheet, which references all icons.

Language Flags

The flags in the language selector are being loaded from Flag Icon CSS. To replace any of the flags, download the package, place the relevant SVG file in the 'system' > 'flags' folder, then update the HTML.

Fade Effect

Template sections fade in when scrolling down the page. To remove this effect, simply delete the 'fade' class from the sections.

Photo panels can also be 'Hero' panels, so that they stretch the entire width and height of the browser window on larger screens. This can be used in conjunction with the carousel code to display full width and height window sliders. For hero panels to work, the code must be added before the booking bar, above the <main> HTML tag. Please note the additional 'hero' and 'full classes' to be added.

Contact Forms

This template includes a fully functional PHP Mailer script and HTML template. The form acts as a booking request, ideal for properties who don't have an online booking engine. The template also includes a newsletter subscribe form, which acts in a similar way.

Hotel Details

Before using the form, you will need to update the PHP code with your details.

// Hotel Details

$hotel_name = "Alpha Hotel";
$hotel_email = "test@klayemorrison.com";

HTML Email Templates

The form uses two templates, found in system/email/ – 'template-guest.php' (sent to users filling out the form) and 'template-hotel.php' (sent to the hotel). Before using, please be sure to update the guest template with your details and imagery.

Calendar Languages

To display the calendar in a different language, go to https://github.com/jquery/jquery-ui/tree/master/ui/i18n, copy the JS from the relevant language file and paste it into global.js before 'datepicker' is called.
